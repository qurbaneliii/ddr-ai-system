# Task interpretation

The original Task Description is the product authority. The target is a clear junior-level production demo: automatic DDR ingestion, native/OCR extraction, deterministic plot processing, normalized SQL persistence, events/summaries/trends/candidates, grounded retrieval chat, and Streamlit delivery.

Provider technologies are implementation choices. OpenAI is optional verbalization; lexical retrieval is the no-key fallback. Deterministic SQL, report text, provenance, and plot evidence remain authoritative. SoR, pressure-time units, cross-namespace identities, and automated anomaly validation remain explicitly unresolved unless supported by source or reviewer evidence.
